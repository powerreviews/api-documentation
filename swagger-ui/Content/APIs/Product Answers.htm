Product Answers

The Write API  ProductAnswers endpoints provide external integration endpoints to PowerReviews ProductAnswers for direct server-to-server integration 3rd parties. These endpoints include:

  • SubmitQuestion - creates a question for a given product or services.
  • SubmitAnswer - creates an answer for a given question.

It's important to note that keys should be used as identifiers for unique items within a specific display and aren't for targeting with our API.

This page contains the following sections:

  • Global Parameters
  • Commonly Used Parameters
  • Endpoints

Global Parameters

The following table lists parameters across the Write API.

Parameter Name Data Type Required? Description
merchant_group_id Numeric Yes The PowerReviews assigned MerchantGroup Identifier.
site_id Text Yes Your internal site identifier. This is required if you use merchant_group_id.
merchant_id Numeric No The PowerReviews unique identifier for a merchant. You can use this parameter instead of merchant_group_id.
api_key Text Yes The token used to secure your requests.
locale Text No The locale used for your API call. If you don't provide this parameter, it defaults to your communication locale.

Commonly Used Parameters

The following table lists commonly used parameters.

Parameter Name Data Type Description
page_id Text The page ID for a particular product. This should be a unique identifier.
page_id_variant Text The variant for a particular product. For example, a shirt with different colors.
f Text The feature flag parameter. Using this parameter you can override the default functionality or functionality provided by merchant properties. You can submit multiple parameters with this name on the same request.

Common Response Attributes

The following are common response attributes for the Write Services API:

  • status_code - numeric parameter.
  • message - text parameter.
  • error_code - text parameter.
  • error_detail - text parameter.

Endpoints

SubmitQuestion

You can find the SubmitQuestion endpoint at the following URL: /api/b2b/question

The request Content Type is application/json.

The request method is POST.

The following table lists the request parameters for the SubmitQuestion endpoint.

Parameter Name Data Type Required? Description
page_id Text Yes Refer to Commonly Used Parameters.
page_id_variant Text No Refer to Commonly Used Parameters.
f Text No Refer to Commonly Used Parameters.

The following table lists the JSON Body Attributes for the SubmitQuestion endpoint.

Parameter Name Data Type Required? Description
question_text Text Yes The text of the question.
iovation_black_box Text Configured The authenticity iOvation blackbox data. You can override if this parameter is required with your merchant property.
merchant_question_category Text No  
question_tye Text Yes The type of question. This parameter supports either service or product as values.
author_name Text Configured The name of the person who asked the question. You can override if this parameter is required with your merchant property.
author_location Text Configured The location of the person who asked the question. You can override if this parameter is required with your merchant property.
author_email Text No The email address of the person who asked the question.
merchant_user_id Text No Your unique identifier assigned to the person who asked the question.
merchant_question_id Text No Your unique identifier for the question. You can use this parameter to create answers for the question.

The JSON response body echoes the request of the JSON body, substituting any server-side adjustments, plus the combination of common response attributes.

SubmitAnswer

You can find the SubmitQuestion endpoint at the following URL: /api/b2b/answer

The request Content Type is application/json.

The request method is POST.

The following table lists the request parameters for the SubmitAnswer endpoint.

Parameter Name Data Type Required? Description
page_id Text Yes Refer to Commonly Used Parameters.
page_id_variant Text No Refer to Commonly Used Parameters.
f Text No Refer to Commonly Used Parameters.

The following table lists the JSON Body Attributes for the SubmitAnswer endpoint.

Parameter Name Data Type Required? Description
question_id Numeric Yes The identifier of the question being answered.
merchant_question_id Text No Your unique identifier for the question. You can use this parameter to create answers for the question.
iovation_black_box Text Configured The authenticity iOvation blackbox data.
answer_text Text Yes The answer text.
author_name Text Configured The name of the person who answered the question. You can override if this parameter is required with your merchant property.
author_location Text Configured The location of the person who answered the question. You can override if this parameter is required with your merchant property.
author_email Text No The email address of the person who asked the question.
merchant_user_id Text No Your unique identifier assigned to the person who asked the question.
answer_source Text No The source of the answer. Accepted values for this parameter include: crowd, web, email, dashboard, and end_or_review.
is_expert Boolean No Indicates whether the person who answered the question is a Subject Matter Expert (SME).
is_verified_buyer Boolean No Indicates whether the person who answered the question is a verified buyer.

The JSON response body echoes the request of the JSON body, substituting any server-side adjustments, plus the combination of common response attributes.

Feature Flags

The feature flags include the skipProfanity flag. You can set this flag to true or false depending if you want to skip the profanity check on content submission.

Only pending questions or answers are accepted by this endpoint.
