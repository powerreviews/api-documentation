Write API

The Write API endpoints provide external integration endpoints to PowerReviews Write-a-Review form for server-to-server implementations. These endpoints include:

  • GetReviewTemplate - this endpoint retrieves the set of review questions for a specific page id, locale, and user ID. This response is fully augmented by local specific content where applicable.
  • SubmitReview - this endpoint submits a user's review data for a product.

These endpoints can be found at: http://writeservices.powerreviews.com/api/b2b/

It's important to note that keys should be used as identifiers for unique items within a specific display and aren't for targeting with our API.

Global Parameters

The following table lists parameters across the Write API.

Parameter Name Data Type Required? Description
merchant_group_id Numeric Yes The PowerReviews assigned MerchantGroup Identifier.
site_id Text Yes Your internal site identifier. This is required if you use merchant_group_id.
merchant_id Numeric No The PowerReviews unique identifier for a merchant. You can use this parameter instead of merchant_group_id.
apikey Text Yes The token used to secure your requests.
locale Text No The locale used for your API call. If you don't provide this parameter, it defaults to your communication locale.

Commonly Used Parameters

The following table lists commonly used parameters.

Parameter Name Data Type Description
page_id Text The page ID for a particular product. This should be a unique identifier.
page_id_variant Text The variant for a particular product. For example, a shirt with different colors.
f Text The feature flag parameter. Using this parameter you can override the default functionality or functionality provided by merchant properties. You can submit multiple parameters with this name on the same request.

Endpoints

GetReviewTemplate

You can find the GetReviewTemplate endpoint at the following URL: /api/b2b/writereview/review_template

The request Content Type is application/json.

The request method is GET.

The following table lists the request parameters for the GetReviewTemplate endpoint.

Parameter Name Data Type Required? Description
page_id Text Yes Refer to Commonly Used Parameters.
page_id_variant Text No Refer to Commonly Used Parameters.
f Text No Refer to Commonly Used Parameters.
merchant_user_email String No Your email address.
String No Your unique identifier that identifies duplicate submissions for a review.

The following table lists the JSON Responses for the GetReviewTemplate endpoint.

Attribute Name Data Type Description
fields JSON object Array

A list of configuration objects that render each form field of the review submission form. Fields can be one of the following types:

  • simple
  • collection
  • composite

The order of a configuration object denotes the general display ordering of fields.
started_timestamp Timestamp The server-generated timestamp that denotes the start of a consumer session.
context_information JSON object The context information regarding the review template used for debugging purposes.

Common Field Types

Attribute Name Data Type Description
field_type Text

An enumerated value, either simple, collection, or composite.
key Text The unique identifier for a review question.
id Text An internal identifier for a field that's provided for debugging purposes. The ID format indicates whether the field is derived from an MSQ or has AttributeGroupOverride applied.
label Text The field label text that displays in the review form.
is_required Boolean A yes/no field that specifies if a consumer response is required for the question.
is_hidden Boolean A yes/no field that specifies if a field should display. In this case, the values of the field are managed by application logic.
helper_text Text A localized prompt of information that displays as part of the element rendering.

Field Type - Simple Field

Attribute Name Data Type Description
value Text/Numeric

The value to populate the field as a default value. The value is numeric if the input_type is stars, otherwise this field is text.
input_type Text An enumerated value. The valid choices for this field are textbox, textarea, and stars.
max_length Numeric The maximum number of characters allowed for text-based input. If this attribute is missing, it's assumed to be unlimited. For the stars input_type, this value denotes the highest rating value allowed.

Field Type - Collection Field

Attribute Name Data Type Description
values Text/Numeric

The value that populates the field as a default value. The value is numeric if the input_type is stars, otherwise this field is text.
choices JSON array The default choices to populate. The maximum supported length of individual values/choices is 100 characters.
input_type Text

Indicates the type of control to render the form element. The supported values include:

  • TextBoxVertical
  • RadioVertical
  • RadioHorizontal
  • RadioHorizontalLinked
  • Dropdown
  • AgeDisclosure
answer_type Text An enumerated value that supports single and multiple values. Certain input_type values imply answer_type.

Field Type - Composite

Attribute Name Data Type Description
input_type Text

An enumerated value to indicate the composite type. Supported values are simple, image, and video.
prompt Text Auxiliary text to display in addition to the field label.
count Numeric Maximum number of times to duplicate the fields input fields as a group.
fields JSON array Similar to fields from the top-level structure, but only simple and collection types are allowed.

SubmitReview

You can find the SubmitReview endpoint at the following URL: /api/b2b/writereview/submit_review

The request Content Type is application/json.

The request method is POST.

The following table lists the request parameters for the SubmitReview endpoint.

Parameter Name Data Type Required? Description
page_id Text Yes Refer to Commonly Used Parameters.
page_id_variant Text No Refer to Commonly Used Parameters.
f Text No Refer to Commonly Used Parameters.
merchant_user_id String No Your merchant id.
merchant_user_email String No Your merchant email address.
String No Your provided identifier to identify duplicate submissions for a review.

The following table lists the JSON Request Body for the SubmitReview endpoint.

Attribute Name Data Type Required? Description
fields JSON Object Yes  
started_timestamp Timestamp No The server defaults to the current time if it isn't provided.
submitted_timestamp Timestamp No The server defaults to the current time if it isn't provided.
reviewer_type Text No The accepted values are site_member and staff.
ip_address Text No The IP address of the person who submitted the review.
iovation_black_box Text No The authenticity iOvation blackbox data.
browser Text No Allows the capture of user-agent string.
promo_code Text No Your specified identifier used to identify any promotion associated with a review.
campaign_id Text No Your specified identifier used to identify any promotion associated with a review. This parameter isn't validated by the PowerReviews platform.
order_id Text No The unique identifier managed by a given merchant, and identifies an order for which the reviewer purchased the product under review.
source Text No Accepted values are web and email. This attribute defaults to web.
disclosure_code Text No Accepted values are sampling and sweepstakes.

Field Type - Common Fields

The following fields are required as part of any field object in the SubmitReview JSON request body.

Attribute Name Data Type Description
field_type Text

Valid values for this field are simple, collection, and composite.
key Text The key that uniquely identifies a review questions.
id Text Your internal identifier for a field that's provided for debugging purposes. The ID format indicates whether the field is derived from an MSQ or has an AttributeGroupOverride applied to it.

Field Type - Simple Field

Attribute Name Data Type Description
value Text/Numeric

The value to populate the field as the default value. This value is numeric if the input_type is stars, otherwise, this value is text.

Field Type - Collection Field

Attribute Name Data Type Description
values Text/Numeric

The value to populate the field as the default value. This value is numeric if the input_type is stars, otherwise, this value is text.

Field Type - Composite Field

Attribute Name Data Type Description

Uniquely identifies a review media object. For example, media_image_0, media_videourl_0.
composite_type Text The type of review media. This currently supports Image and Video.
composite_data JSON Object

Response

If a request fails to pass validation, a response similar to the GetReviewTemplate query returns. However, only fields that failed validation return and have the error_message attribute populated.

Field Type - Common Fields

Attribute Name Data Type Description
error_message JSON object Error information that displays for a given field, if the field value is in error.

Error Message

This object can be top-level or nested inside of a field object.

Attribute Name Data Type Description
code Text The unique identifier describing the error. This attribute is different than status code.
message Text A localized message that displays to an end user.
detail JSON object Additional details about the error used for the programmatic handling of the error.

Common Error Responses

The following table lists error responses that reject a request in its entirety.

Error Code Http Status Description

Missing Merchant 400 Missing parameter combination to identify the merchant account.
MissingParameter 400 General error message if a required parameter is missing. The message identifies the missing parameter.
Invalid Parameter 400 General error message if a parameter fails validation. The message identifies the missing parameter.
CouldNotSaveImage 400  
ImageUploadNotEnabled 400  
UnrecognizedField 400 The field provided has a key that isn't a part of the expected review template.
ValidationFailure 400 One of the fields submitted failed validation.
MerchantNotEnabled 401  
MerchantNotEnabledForSSL 401  
Missing API 401  
Invalid API Key 403  
ReviewAlreadySubmitted 403 Returns if t he review is identified as already submitted.
ReviewTooLong 413 Returns if the review content was too large for the internal system to handle.
InternalError 500 Returns if unexpected error is encountered in the server.

Feature Flags

Feature Flags Description
Profanity Check skipProfanityCheck Allows profanity checks to be skipped on content submission.

Additional Details

Only pending status reviews are accepted by these endpoints.

Stored reviews have audit information stored in the REVIEW_EXTRA table if the review has a flag denoting that the review was via the API endpoint.

Content submission from high-volume merchants is isolated from general online submission and that of low-volume merchants.
